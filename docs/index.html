Adding people to the discuss@ listserv

Anybody can subscribe themselves to the discuss listserv, but only existing staff can add others to discuss.

  • go to the somervillebikekitchen.org admin console:
    https://admin.google.com/somervillebikekitchen.org/AdminHome
  • login as "admin@somervillebikekitchen.org" (obtain the password from another staff member)
  • click on Groups
  • click on the discuss listserv
  • click on Manage users in discuss
  • paste the e-mail addresses in the text box (separate each address with a comma)
  • select Member from the drop-down and click Add

Adding new staff to the staff@ listserv

Only existing staff can add others to staff@.

  • go to the somervillebikekitchen.org admin console:
    https://admin.google.com/somervillebikekitchen.org/AdminHome
  • login as "admin@somervillebikekitchen.org" (obtain the password from another staff member if you cannot remember)
  • click on Groups
  • click on the staff listserv
  • click on Manage users in staff
  • paste the e-mail address in the text box
  • select Owner from the drop-down and click Add

Staff onboarding

  • add a new column to the sheduling spreadsheet
  • subscribe the new staff member to the discuss listserv as a "Member"
  • subscribe the new staff member to the staff listserv as a "Member"
  • record the new staff member's information in a physical list inside the shop
    • full name
    • e-mail address
    • alternative contact
  • Make a copy of the Sprout key and give it to the new staff member.
  • Email the new staff member with something similar to the following text: 
    Hi new staff member!

Please familiarize yourself with the staff page:

http://somervillebikekitchen.org/docs/staff.html

This is where you can find all the information needed by staff members.  Look
there to find the latest schedule and various instructions.

Please sign up for your first open shop by looking for an unscheduled day when
you are available, and requesting to host it (send mail to staff@).

Thanks, and see you soon!

Organizing events

Events increase popularity and, indirectly, membership. They facilitate interaction between members of the local cycling community. Generally, there are two types of events: social events and workshops. If you are trying to organize an SBK event, see instructions below:

  1. Identify the person who will run the event, or the presenter. Work with the presenter to decide on the best time(s) for the event. The event may be multipart (span across multiple days, e.g. introduction session, dirty hands session, advanced topics session).
  2. Write short descriptions (short paragraph) for each session/day of the event.
  3. [optional] Create a flyer for the event. EXAMPLE FLYER
  4. Add the event to the SBK website. Make a new event page for each session/day of the event. INSTRUCTIONS. Contact Troy for help.
  5. Send an announcement e-mail to the discuss@ mailing list. Aggregate the descriptions for all sessions/days of the event and send one e-mail. Touch on the following topics: introduction, rationale, sessions, rsvp request (usually informal), admission fee (usually free with suggested donation of $10), and any help needed from the community (e.g. we may need to borrow tools). Remember to attach the flyer, if there is one.
  6. [optional] Re-send the announcement e-mail to other mailing lists as you see fit (do not include multiple lists in the same e-mail). Use your best judgement to determine which additional mailing lists will find this event relevant.
  7. [optional] Create a facebook event. Remember to attach the flyer, if there is one. Contact Val for help.
  8. [optional] Create a twitter post. Remember to attach the flyer, if there is one. Contact Jenny for help (search your inbox for Jenny's e-mail address).

Editing the website

Prerequisites: Git, Make, Jekyll. On Debian/Ubuntu, you can install the prerequisites using this command:

sudo apt-get install git make jekyll

Follow these steps to edit the website:

  1. clone the website using git:
    git clone https://github.com/somervillebikekitchen/somervillebikekitchen.github.io.git
    or if you've already cloned before, make sure you have the latest version:
    cd path/to/sbk
    git pull
  2. make your desired changes
  3. spin up test server:
    make run
  4. navigate to http://localhost:8000/ in your web browser
  5. make sure your changes look correct
  6. commit your changes:
    git commit -am "Your reason for change"
  7. If you have a github account and you've been added to the repository as a collaborator, you can just push and instantly update the website:
    git push
